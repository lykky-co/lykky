How should we render the ecosystem map?

Each approach groups organizations by function (Funding, Programs, Workspaces, etc.) as a network diagram. Pick the one that feels right for the site.

A

SVG Node Map (Static + Hover)

Hand-crafted SVG layout with nodes grouped in clusters by category. Hover reveals details. No external libraries. Lightweight, fast, works everywhere. Think: a clean infographic-style map.

Pros

  • Zero dependencies
  • Fast load, accessible
  • Full design control
  • Matches dark theme easily

Cons

  • Layout is manual — adding nodes means repositioning
  • Not draggable/zoomable without extra work
B

D3.js Force-Directed Graph

Physics-based layout where nodes repel and category links pull them together. Interactive — drag nodes, zoom, pan. Nodes cluster naturally by function. Classic network visualization.

Pros

  • Auto-layout — adding nodes just works
  • Interactive (drag, zoom, pan)
  • Visually dynamic

Cons

  • ~50KB dependency (d3-force)
  • Physics can feel jittery
  • Harder to make look polished
C

CSS Grid Cluster Map

Pure HTML/CSS — each category is a visual cluster (card group) with nodes inside. Lines drawn with SVG overlay connecting related orgs. Clean, readable, responsive. More like an interactive org chart than a traditional network graph.

Pros

  • Zero dependencies
  • Fully responsive
  • Most readable / accessible
  • Easy to maintain — just edit data

Cons

  • Less "map-like", more structured
  • Connection lines between clusters are tricky on resize
